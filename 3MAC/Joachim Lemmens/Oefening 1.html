Text 12 Oct Week 3, les 1 - Zelf webpagina’s aanpassen

Na vorige week de basis van het maken van een HTML-code te hebben gezien, gingen we deze week verder op deze basis. We kopieerden namelijk HTML-codes uit webpagina’s en pasten hier zelf een beetje deze codes in aan.

Eerst een kleine theoretische inleiding.
Het World Wide Web bestaat uit webstandaarden, deze standaarden zijn ontworpen voor ondermeer HTML, XML en CSS. De organisatie die deze standaarden ontwerpt, noemt World Wide Web Consortium (W3C).

Er bestaat ook de defactostandaard, dit is een protocol of een programmeertaal die als standaard wordt beschouwd, maar echter niet door een ‘standaard’organisatie is goedgekeurd. Deze standaard is echter wel veelgebruikt binnen de bepaalde industrie.

We kennen bijvoorbeeld Windows. Vroeger bepaalde Windows zelf de standaard, ze hadden toen het grootste marktaandeel (nu trouwens nog betreffende Desktoppen, maar in mindere mate).
Nu is Apple het ‘nieuwe’ Windows, zij volgen niet de standaarden op de markt. Op de markt van de Smartphones is Apple dan ook marktleider.
Elke marktspeler werkte samen aan een betere ‘open source’ standaard en voegde er zelf elementen aan toe. Apple voegde er echter elementen aan toe die andere niet konden gebruiken.

Webpagina’s zelf aanpassen


Niemand kan je website op het internet zien voor je hem hebt gedeployed, voor een webpagina noemt dit publiceren.
Als we een HTML-code van een website willen zien drukken we op de rechtermuisknop en vervolgens selecteren we Paginabron weergeven. Omdat dit van browser tot browser verschilt, hebben we besloten op Google Chrome te gebruiken.

Nu gaan we enkele elementen van de HTML-code aanpassen, hiermee willen we aanpassingen aan de webpagina maken, die door de browser niet standaard gebeuren. Dit is namelijk het doel van Webdesign. We willen onze eigen webpagina’s verbeteren!
We pasten bv. het h1 element in de <body> aan.

h1{font-weight:normal;}

-> Door dit te doen werd ondermeer het vet van de titel verwijderd.

<!—xxx —> = hulpstuk, De tekst binnen deze tekens zal niet op je webpagina verschijnen.
/*Stijldefinities */ = Stijldefinities zal bv. Niet worden weergegeven/toegepast op je webdocument.
figure img { max-width: 100%;} = figuur word kleiner als je de browser ook kleiner maakt. De figuur mag Max. 100% breed van browser zijn.
