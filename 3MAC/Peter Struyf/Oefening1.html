Les 2

We hebben op zijn minst drie websites; Blog Facebook Twitter We gaan leren werken met een notepad om te coderen. Door tools is het makkelijker om te coderen, vroeger was het veel moeilijker. Het internet; Het internet is een geheeld van interconnected computer networks. Computer bestaat uit moederbord, CPU (rekenmachine); Berekeningen doen op die date. Die krijgt dat binnen van een geheugen (RAM). Die stuurt dat naar de CPU (X Y + = resultaat Z) U computer is een netwerk van chips. (GPU, CPU, Netwerk, RAM, SSD, HD, …) Dat zit verbonden via kabels en notes. Basisprotocollen = afspraken hoe computers met elkaar communiceren. (TCP/IP) Hoe gaan die data uitgewisseld worden? Intranet = lokaal netwerk. (Bv school) De ‘baas’ van het netwerk controleert alle computers van het intranet, een computer in de kelder. Deze controleert de toegang tot het internet. Deze computer is ook een note in het netwerk van Telenet/Belgacom. Het internet = verzameling van alle intranets van de wereld, steeds verbonden via de note, de gast, de host. Ad hoc netwerk = een netwerk via kabels. Gsm is via een mast, en die mast heeft rechtstreeks connectie met Mechelen. In Stockholm bellen de mensen zonder GSM masten, maar door een adhoc netwerk via elkaars gsm. Er is geen baas van het internet. In praktijk is er een baas over lokaal internet (Belgacom, KHLeuven, …) De TOEGANG is wel bepaald door mensen. Elke computer heeft een nummer, en dat wordt bijgehouden door een non-profit in SF. Gedecentraliseerd; Als Telenet gehackt wordt, is het internet niet stuk, dat kan niet. Browser = bladeren door websites. Browser kan die lezen en weergeven volgens bepaalde protocollen (html). Het world wide web is maar een klein deel van het internet. Wat gebeurt er op het internet; - Publicatie van documenten (Html, webpagina’s, web apps (programma in de webbrowser)). - data transfer (data, files; ftp, APIs) - Communicatie (VoIP zoals skype, IM Instant Messenger zoals MSN,…) - Inter-linked hypertext documents (http, https) - file transfers (ftp) - e-mail (eigen protocol) - peer-to-peer networks (p2p; voorbeeld Bittorrent) Steeds nieuwe protocollen voor nieuwe media, … Chrome, Safari gebruiken dezelfde rendering engine; Webkit (kan html begrijpen en renderen). Open Scource; iedereen kan dat gebruiken. Web client = Programma dat een rendering engine bevat. Web page kan nog zaken bijhouden zoals bookmarks…

Les 4

De verenigde naties van het internet W3C, is een organisatie die bestaat uit een aantal partijen die afspreken hoe het internet werkt. Hoe werkt HTML, URL, hoe moet dat opgesteld worden etc. Die afspraken = webstandaarden. Je kan deze volgen of niet. Webkit browser van chrome en apple volgen deze goed. Internet explorer niet. Dit omdat Microsoft het grootste marktandeel had. Microsoft bepaalde de standaard met 80%, maar nu niet meer dus moet Microsoft meer volgen. Microsoft is nog altijd de grootste in desktop pc’s, maar dit daalt. Nu begint Apple te zeggen; wij zijn de baas en wij doen onze eigen zin met standaarden van het internet. De Webkit en Mozzilla rendering engine is open scource. Om zich te beschermen tegen de grote bedrijven zoals Apple en Microsoft, maken Webkit en Mozzilla hun engine Open Scource. Apple voegde zaken toe aan webkit maar publiceerde dit niet als Open ScourceGoogle heeft Webkit ‘geforked’: -> Wel zelf zaken aan toegevoegd, maar wél Open Scrouce gemaakt. Samenvatting; Internet leeft ook in een economische realiteit en zal altijd zo blijven. Grote bedrijven zullen elkaar altijd beconcurreren en pesten. Uitrollen van website; Op server zetten. Index.html gemaakt en aangepast BrowserDefaults; Alles wat de browser zelf kiest als er geen stijlen zijn gezet in het html document. Wij als webdesigners gaan andere zaken schrijven dat het niet zo saai en lelijk is. Alles tussen negeert je browser, dit is voor jezelf als herinnering of … Alles tussen /* *\ negeert de browser ook, als je deze wegdoet dan zal je browser dit gaan toepassen. Opletten bij het schrijven van een pagina; nooit fel wit op zwart doen, dat geeft koppijn bij het lezen van de tekst. Vb: 253 ipv 255. Om een afbeelding helemaal op een scherm te laten, en dus niet te laten cutten als je scherm verkleint, is het belangrijk om de lijn figure img { max-width: 100%;} in te voegen. -> wordt uitgevoerd op voorwaarde dat ons image-element in een figure-element zit. Als we zeggen dat de afbeelding max 100% mag zijn, gaat de browser op zoek gaan naar het eerste parent element dat een breedte heeft gezet. Dit is de breedte van het scherm van de browser zelf. CSS = Cascading Style Sheet. HTML heeft een hiërarchische structuur. Het heeft dus een topelement waar andere van aftakken. CSS werkt als een waterval.

Les 6

In deze les hebben we het document index.html van vorige keer (datgene dat over geschiedenis ging) helemaal eigen gemaakt. We hebben onze blog gekopieerd naar deze index.html en bewerkt zodat het er beter uitziet dan de standaard-html stijl. We leerden dat we via ‘element inspecteren’, elke soort opmaak kunnen bestuderen en dan toepassen in onze eigen blog. Het resultaat is dat onze blog niet zomaar een blog is online zoals tumblr of wordpress (met hun stijl en vormgeving), maar een eigen blog met eigen vormgeving en stijl. Deze blog staat nog niet online omdat daar een server voor nodig is. Momenteel is de index.html enkel te bezien op eigen pc of in het lokale netwerk van de school. Mijn persoonlijke mening over deze les is dat het enorm handig is om html-codes te leren gebruiken. Het ziet er allemaal enorm ingewikkeld uit, maar na een kleine uitleg werd het al snel duidelijk dat er veel logica inzit. Door de hulp van W3schools.com kunnen we ook veel zelf opzoeken en toepassen op onze eigen blog.
