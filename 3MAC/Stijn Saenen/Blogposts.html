Webdesign en DTP: Les 3

Gemaakt door Stijn - 15 oktober 2013

HTML

HTML (Hypertext Markup Language) is een soort programmeertaal waarmee documenten opgemaakt kunnen worden, en bestaat uit platte tekst. Dit wordt voornamelijk gebruikt voor opmaak op een internetpagina. Html is een basistaal waarin andere talen (zoals javascript, php,...) in geïmplementeerd kunnen worden.

Het werken met html zit redelijk logisch in elkaar, maar is in het begin nogal ingewikkeld om zelf te doen. Zo heeft elk type opmaak zijn eigen soort commando, ook wel tags genoemd. Door die tags wordt er aangegeven hoe platte tekst moet worden geïnterpreteerd. Zo heeft 'paragraaf' de tag '<p>'. Wanneer de gebruiker een paragraaf wil invoeren, moet hij voordat de tekst wordt ingegeven de paragraaftag eerst openen. Dit kan door <p> vooraan te zetten. Wanneer de gebruiker de paragraaf wil afsluiten, moet hij/zij </p> achter het bericht zetten. Door de schuine streep wordt de tag afgesloten en voert html de tag uit.
Dit is een voorbeeld van de <ul>- en <li>-tags:

Deze afbeelding geeft weer hoe die tags precies werken. <ul> staat voor 'Unordered list'. Door deze tag te gebruiken weet je dat je een lijst van iets gaat maken. Achteraan wordt de list gesloten met de tag </ul>. Tussen deze tags staat er platte tekst, met vooraan en achteraan <li> tags. <li> staat voor 'List item' dus wanneer deze tag geopend en achteraf gesloten hebt, weet je dat, wat er tussen de tags staat, weergegeven zal worden als een bullet IN je lijst:

Tags werken ook van buiten naar binnen. Je kan dus binnen tags ook andere tags gebruiken, maar ze moeten op de juiste volgorde staan. Anders kan het zijn dat ze niet werken, of het kan leiden tot bugs in de website.

Er zijn zeer veel verschillende tags, en door de jaren heen zijn er ook zeer veel andere bijgekomen. Aangezien iedereen programmeert hoe die persoon dat zelf wil, zijn er ook geen regels opgesteld. Dit kan in webdesign soms ook wat chaos met zich meebrengen. De ene persoon codeert op deze manier, de andere op een andere,...
Om dit deels op te lossen heeft het World Wide Web Consortium (W3C) richtlijnen (webstandaarden) opgesteld waarin programmeurs zich het beste houden. Dit is niet verplicht om te doen, maar wel aan te raden. Zo krijgen html pagina's een beetje dezelfde structuur. Wanneer je een webpagina gemaakt hebt, kan je deze laten controleren door het W3C. Zij hebben een tool waarin je de website kan ingeven en zij gaan na of deze geschreven is volgens hun regels. Als dat niet is, worden de fout geschreven regels getoond en kan dit (naar eigen keuze) aangepast worden voor jezelf. Ook wanneer je een fout hebt geschreven en je kan ze niet terugvinden in de code, kan het zijn dat deze tool van het W3C de fout kan opsporen en je exact kan zeggen waaraan het ligt.

Persoonlijk vind ik dit een zeer handige tool en ik vind ook dat alle sites zich best aan deze webstandaarden houden. Dit zorgt voor meer organisatie en overzicht voor programmeurs, maar ook voor de visualisatie op het internet

Dit is een duidelijk filmpje dat de beginselen van HTML uitlegt.

Hier wordt ook in uitgelegd waarom het gebruik van Dreamweaver niet zo een goede keuze is.

CSS

Een ander veelbesproken punt tijdens deze lessen, is CSS (Cascading Style Sheet)

CSS zorgt voor de vormgeving en de opmaak van de webpagina en kan geïmplementeerd worden in HTML. Het zorgt dus (deels) voor een mooiere weergave. Met CSS kunnen er bijvoorbeeld kleuren aangepast worden, maar ook opties voor afbeeldingen (sizing, resizing), tekstterugloop, enzovoort. Bepaalde HTML tags kunnen via CSS ook een aangepaste lay-out aannemen. Zo kan je bijvoorbeeld zeggen in CSS: alle onderdelen die de <p>-tag hebben op deze webpagina, moeten blauw weergegeven worden. Zulke opties geeft een beter gebruiksgemak aan de webdesigner, zonder constant te moeten kijken of de lay-out is zoals de webdesigner dat wil.

Ook voor CSS heeft het W3C een standaard vastgelegd omdat, net zoals bij HTML, CSS snel vooruit is gegaan op enkele jaren tijd met meer en meer features. Het W3C past hiervoor hetzelfde controlesysteem toe als bij HTML, waar gebruikers van de tool kunnen zien of ze fouten in de code hebben of zich niet houden aan de standaarden.
Een CSS code kan geïmplementeerd worden in een HTML code, maar die kan ook evengoed opgenomen worden in een ander bestand. Dit bestand kan dan gelinkt worden aan een pagina. Het voordeel van een apart bestand is bijvoorbeeld wanneer je een bepaalde CSS ook wil gebruiken op andere webpagina's. Dan moet de webpagina alleen maar gelinkt worden met het bestand. Het nadeel is dan ook weer dat, wanneer het bestand (al dan niet per ongeluk) wordt verwijderd, je pagina niet meer aangepast is aan die CSS, omdat het weg is. Bij de opkomst van de personal computer nam Microsoft het niet zo nauw met de internetstandaarden. Aangezien zij toen de grootste leiders waren op gebied van computers en internettechnologie, dachten zij dat zij de standaarden mochten bepalen (defactostandaard). Dit was niet goed en Apple maakt stilaan ook dezelfde fout als Microsoft vroeger. Zij houden zich ook niet aan de webstandaarden en drijven hun eigen wil door.

Mijn conclusie is dat HTML en CSS zeer belangrijke bouwstenen zijn. Niet alleen voor het maken van een website, maar ook gewoon om te weten hoe zulke dingen werken. Voor mensen die vaak bezig zijn met tekstverwerking op alle manieren op een computer, kan dit van pas komen.

Webdesign en DTP: Les 2

Gemaakt door Stijn - 7 oktober 2013

Wat komt er allemaal kijken bij het maken van een website?

In deze les werd ons meer uitleg en inzicht gegeven over wat het begrip webdesign allemaal precies inhoudt.
Webdesign is zeer uitgebreid en omvat veel verschillende begrippen. Je kan bijvoorbeeld statische, maar ook dynamische websites aanmaken die bijvoorbeeld gekoppeld zijn aan databanken, of die programmeertalen gebruiken om bepaalde mogelijkheden aan de website te geven. Een dynamische website is dan ook veel gebruiksvriendelijker. Bij deze soort is er bijvoorbeeld een (al dan niet zelfgeschreven) CMS (content management system). Hiermee is de site veel gebruiksvriendelijker voor de persoon die deze up-to-date moet houden. Er zijn ook kant en klare CMS's die gebruikt kunnen worden voor je eigen site, zoals bijvoorbeeld WordPress en Joomla.
Een statische website updaten kan dan weer moeilijker en onoverzichtelijk zijn, wat maakt dat de website er minder professioneel uitziet.
Een ander punt dat werd aangehaald in de les, is hoe een LAN (local area network) werkt. Een LAN is een netwerk van minstens 2 computers die met elkaar verbonden zijn en zo dus met elkaar kunnen 'communiceren'. Er zijn ook nog veel andere types netwerken die worden gebruikt. Zelf ken ik er buiten LAN nog drie andere: PAN (Personal Area Network), MAN (Metropolitan Area Network) en WAN (Wide Area Network)

  • PAN: Wordt gebruikt voor communicatie tussen computer apparaten
  • MAN: Netwerken over een hele stad of een groot gebied
  • WAN: Netwerken over een land of een continent

Soms kunnen er zich problemen voordoen met onder andere de veiligheid van netwerken. Daarom hebben ze protocollen opgesteld die er voor zorgen dat er een optimale communicatie is tussen de verbonden computers. Zo hebben onder andere verschillende soorten media, software en toepassingen bepaalde protocollen: internet, skype (Voice over Internet Protocol), e-mail (POP, IMAP,...). Wanneer er zulke nieuwe tools en mogelijkheden worden ontwikkeld, bestaat de kans dat er ook steeds nieuwe protocollen uitgevonden worden.

Vervolgens werd er in de les overgegaan naar servers. Dit was een goede overgang, want door te weten wat een netwerk is, is het makkelijker om uit te leggen wat een server doet. In het geval van de school, beheert de server de (verschillende) netwerken (intranet) van de school. De schoolserver is op zijn beurt dan weer verbonden met een server van de ISP (Internet Service Provider). De ISP zorgt ervoor dat de school toegang heeft tot het internet, wat een verzameling is van veel intranetten over de wereld.

Veel mensen denken waarschijnlijk dat het niet echt nuttig is om dit te weten voor een vak als webdesign. Maar ik vind wel dat deze uitleg er toe doet wanneer je begint/bezig bent met webdesign. Er komt veel meer kijken bij het online zetten van een echte 'from scratch' geschreven website dan dat de meeste mensen denken. Zo moet je bijvoorbeeld weten waar alles wordt opgeslagen, hoe alles werkt, een webhost vinden die extensies en webruimte aanbiedt,… Bij dit laatste moet je rekening houden met de uptime van de servers van dat bedrijf. Hoe beter de service en servers van de provider van extensies en webruimte, hoe constanter de uptime van je eigen website.

Tijdens de les werd er ook uitleg gegeven over oa. webkit, rendering engines, werken met html, css, enzovoort. Meer hierover volgt in mijn derde blogpost.

Enkele begrippen:

IP: internet protocol

Ad hoc netwerk: een netwerk dat niet moet verbinden met een mast, maar alle (bv. gsms) geven elkaar signaal.

Internet: verzameling van alle intranetjes ter wereld die met elkaar verbonden zijn

HTTP: hypertext transfer protocol

ARPA:Advanced Research Projects Agency: heeft vroeger heel de communicatie-infrastructuur herbekeken om 'backups' te hebben voor communicatie in Amerika

DoS: Denial of Service: Een situatie waarin de computer weigert te werken op de correcte manier. In de meeste gevallen komt dit door een hacker die een systeem wil platleggen, maar in sommige gevallen kan dit ook voorkomen door een bug in het systeem.

Webdesign en DTP: Les 1

Gemaakt door Stijn - 24 september 2013

Deze eerste les ging vooral over hoe alles ging verlopen naar de toekomst toe en wat er van ons wordt verwacht. Eén van de eerste punten waarover gesproken werd, is het gebruik van social media (Twitter) in plaats van e-mail. Dit werd uitvoerig besproken en er was redelijk veel tegenspraak. Veel mensen vragen zich af waarom Twitter beter kan worden gebruikt dan e-mail, omdat het lijkt dat een platform als Twitter minder krachtig en meer beperkt is. Dit vind ik zelf niet waar. Dit platform is zeer krachtig en heel veel bedrijven gebruiken Twitter als brug tussen hun en de klanten, collega's en partners. Er worden stilaan zelfs bedrijven opgericht om het gebruik van Twitter te vergemakkelijken.
Een verre kennis (en ex-schoolgenoot) van mij is ook op deze kar gesprongen. Hij heeft een tool ontwikkeld die hij verhuurt aan bedrijven, waarmee zij overzichtelijk en heel georganiseerd alle Tweets omtrent hun bedrijf van klanten op te volgen. Deze tool werd alsmaar bekender en heeft nu klanten als Volkswagen, Brussels Airport, Mobile Vikings, enzovoort. De oprichter is investeerders gaan zoeken en heeft zo al meer dan 1 miljoen dollar bijeengekregen om zijn tool verder te ontwikkelen.

Sparkcentral (zijn bedrijf) situeert zich in Silicon Valley en groeit steeds meer.
Dit toont aan dat Twitter een waardige concurrent is voor e-mails op professioneel (en privé) gebied. Spijtig genoeg werkt dit alleen goed als je veel volgers hebt. Het zal een tijd duren voordat iedereen verder denkt dan het versturen van e-mails.

Andere punten die werden aangehaald, zijn bijvoorbeeld:

  • Responsive optimization (websites optimaliseren voor verschillende soorten hardware, zoals tablets, telefoons,...)
  • Usability engineering (gebruiksvriendelijkheid van websites en programma's)
  • Professioneel omgaan met tools als Twitter
  • Desktoppublishing
  • HTML, CSS, Javascript.

Dit laatste punt zal meer uitgebreid besproken worden in les 2.
